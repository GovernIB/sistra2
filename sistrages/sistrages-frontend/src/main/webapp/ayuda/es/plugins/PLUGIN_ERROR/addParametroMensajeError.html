addParametroMensajeError(String parametro):void

DESCRIPCIÓN

  • Método que permite añadir el valor de los parámetros a sustituir en el mensaje de error. De esta manera se puede dotar de dinamismo al mensaje de error.
  • Los parámetros se identifican en el mensaje de error mediante llaves ({0} {1} ...).
  • Si el mensaje tiene más de un parámetro, se ha de tener presente que importa el orden en que se añaden los mismos.
  • Este método está disponible en PLUGIN_ERROR, desde todos los scripts de versión, paso y de formulario.

PARÁMETROS DE ENTRADA

  • parámetro: String con el valor del parámetro a fijar.

EJEMPLOS

Pongamos por caso que se quiere controlar la disponibilidad de datos para un determinado NIF en una fecha concreta. Esta operación se podría realizar recuperando datos de un dominio y mostrando un mensaje de error en caso de no existir datos. A continuación se describe cómo se tendría que proceder para parametrizar el mensaje de error.

Previamente, en la pantalla de script se ha de definir un mensaje de error en la pestaña de mensajes validación, por ejemplo con el identificador CodErrFechas y el valor siguiente:
El DNI {0} no tiene datos asociados a fecha {1}.
Desde el script, se hace referencia a ese mensaje de la siguiente forma:

PLUGIN_ERROR.setCodigoMensajeError('CodErrFechas');
PLUGIN_ERROR.addParametroMensajeError('12345678T');
PLUGIN_ERROR.addParametroMensajeError('10/09/2012');

Como resultado de la ejecución del script se obtendrá el siguiente resultado:
El DNI 12345678T no tiene datos asociados a fecha 10/09/2012.
