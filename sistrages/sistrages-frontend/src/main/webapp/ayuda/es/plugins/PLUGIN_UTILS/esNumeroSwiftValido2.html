esNumeroSwiftValido(String banco, String pais, String localidad, String sucursal): boolean

DESCRIPCIÓN

  • Valida que el valor introducido no sea nulo y cumpla con las reglas de validación necesarias para ser un SWIFT (código de identificación bancaria para realizar transferencias) Válido. Se considera válida toda aquella cadena que contenga 4 caracteres del banco, 2 caracteres para el pais, 2 caracteres para la localidad y 3 caracteres para la sucural (la sucursal puede ser nulo y en dicho caso, será sede central). El código del banco y del país debe ser sólo mayúsculas, el código de lugar debe ser sólo mayúsculas o sólo números mientras que el código de sucursal puede ser mayúsculas y números.

PARÁMETROS DE ENTRADA

  • banco: Código de 4 caracteres del banco.
  • pais: Código de 2 caracteres para el país (según la ISO 3166-1-alfa-2).
  • localidad: Código de 2 caracteres para la localidad.
  • sucursal: Código de 3 caracteres de la sucursal.

PARÁMETROS DE SALIDA

  • boolean: true si banco+pais+localidad+sucursal forman un código SWIFT válido y false si banco+pais+localidad+sucursal no forman un número SWIFT válido.

EJEMPLOS

Ejemplo 1 Validar los siguientes valores: banco: 'BBVA', pais: 'ESP', localidad: '88', sucursal: 'XXX'.

var valido = PLUGIN_UTILS.esNumeroSwiftValido('BBVA', 'ESP', '88', 'XXX');

La función esNumeroSwiftValido retorna 'false' ya que los parámetros introducidos no forman un código SWIFT válido, el código de país debe tener longitud 2.


Ejemplo 2 Validar los siguientes valores: banco: 'BBVA', pais: 'ES', localidad: '88', sucursal: 'XXX'.

var valido = PLUGIN_UTILS.esNumeroSwiftValido ('BBVA', 'ESP', '88', 'XXX');

La función esNumeroSwiftValido retorna 'true', ya que los parámetros introducidos forman un código SWIFT válido.
