addAnexo(String identificador, String descripcion, String extensiones, String tamanyoMax, String urlPlantilla, boolean obligatorio, boolean convertirPDF, boolean firmar): void

DESCRIPCIÓN

  • Método que permite añadir un nuevo anexo de forma dinámica a la versión del trámite.

PARÁMETROS DE ENTRADA

  • identificador: Identificador del anexo.
  • descripcion: Descripción del anexo.
  • extensiones: Extensiones permitidas para el anexo. Lista de valores separados por comas. Es necesario indicar al menos una extensión.
  • tamanyoMax: Tamaño máximo del anexo. Es opcional. Si se indica, se debe añadir la unidad al final del mismo. Valores posibles:
    1. KB
    2. MB
  • urlPlantilla: Url en la que se puede descargar la plantilla del anexo. Dato opcional.
  • obligatorio: Variable booleana que permite marcar si el anexo es obligatorio o no.
  • convertirPDF: Variable booleana que permite marcar si el anexo se debe convertir a pdf.
  • firmar: Variable booleana que permite marcar si se debe firmar o no el anexo.

EJEMPLOS

Si deseamos añadir un nuevo anexo de manera dinámica a la versión de un trámite, ejecutaríamos lo siguiente:

DATOS_ANEXOS_DINAMICOS.addAnexo('ANE1', 'Documento de conformidad', 'doc,pdf', '1024 KB', 'http://caib.es/anexos/plantillaConformodidad.pdf', true, true, true);

El resultado de la ejecución del script "Lista dinámica de anexos" con esta sentencia será un anexo que es obligatorio, que debe subirse en formato doc/pdf, con un tamaño máximo de 1024 KB (1MB), que ha de ser firmado y que se convertirá a pdf.
