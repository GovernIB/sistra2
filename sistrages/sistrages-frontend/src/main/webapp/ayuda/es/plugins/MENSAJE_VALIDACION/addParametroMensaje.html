addParametroMensaje(String parametro): void

DESCRIPCIÓN

  • Método que permite establecer el valor del parámetro a sustituir en el mensaje. Esta funcionalidad permite establecer contenido dinámico en los mensajes de validación.
  • Los mensajes se definen en la pestaña de mensajes de validación con un identificador.
  • Dentro del mensaje se definen los parámetros mediante números entre llaves, pudiendo tener más de uno. P.e. {0} {1}...
  • Si el mensaje tiene más de un parámetro, se debe tener en cuenta que importa el orden de los mismos a la hora de sustituirlos.
  • Este método está disponible desde el método PLUGIN_MENSAJEVALIDACION.crearMensajeValidacion (accesible desde todos los scripts de versión, paso y formulario).

PARÁMETROS DE SALIDA

  • parametro: String con el valor del parámetro a sustituir.

EJEMPLOS

Pongamos por caso que queremos fijar el valor de los parámetros de un mensaje de validación, por ejemplo MENSAJE_001, previamente creado. Los pasos a seguir serían los siguientes:

En la pantalla de script se define, en la pestaña de mensajes validación, el mensaje con identificador MENSAJE_001 y dos parámetros mediante el texto:

Es necesario que se lleve a cabo un debug con los parámetros {0} y {1}.

Desde el script se referencia dicho mensaje de la siguiente forma:

var msg = PLUGIN_MENSAJEVALIDACION.crearMensajeValidacion('MENSAJE_001');

posteriormente se le añaden los parámetros 0 y 1:

msg.addParametroMensaje('param 0'); msg.addParametroMensaje('param 1');

de manera que se sustituirá las cadenas {0} y {1} por param0 y param1 respectivamente.
