DISEÑO DEL FORMULARIO

En la pantalla de 'Diseño del formulario' se puede diseñar y previsualizar formularios asociados a la versión del trámite.
Esta pantalla se compone de una cabecera, en la que se presentan las propiedads de formulario y la barra de diseño de componentes, un área en la zona izquierda en la que se presenta un editor visual del formulario, un área derecha de configuración de propiedades de los campos, y un pie compuesto por una barra de botones con diferente funcionalidad.
Cualquier cambio realizado sobre los campos en edición deberán aplicarse mediante el botón 'Aplicar cambios' para que se visualicen en el área de edición

BARRA DE DISEÑO

  • Formulario: pulsando este botón se abrirá la ficha para configurar las propiedades del formulario (cabecera, gestión de páginas y plantillas de visualización).
  • Componentes: sección que presenta los botones para la inserción de componentes en el formulario. Los tipos de campos disponibles son:
    • Línea: este botón permite incluir una línea en el formulario. Cada componente deberá posicionarse en una línea.
    • Sección: este botón permite crear un campo sección. Este tipo de campos permite marcar los diferentes apartados o secciones de un formulario.
    • Aviso: este botón permite incluir avisos informativos de diferentes tipos (info, warning y error). Cada tipo de aviso presenta una iconografía ilustrativa.
    • Campo de texto: este botón permite incluir un campo de texto.
    • Campo selector: este botón permite incluir un campo selector. Este campo permite presentar diferentes valores y permitir la selección de uno o varios valores al ciudadano, dependiendo del tipo de selector (desplegable, selección múltiple o selección única). En caso de utilizar la selección única en horizontal, hay que ocupar suficientes columnas para poder tener la vista en horizontal. En caso de que no se dejen, en la vista veremos las opciones en vertical.
    • Campo checkbox: este botón permite incluir en el formulario un campo casilla de verificación.
  • Posicionamiento: sección que permite especificar la posición que se asignará a un campo al crearlo respecto a un campo seleccionado. Los valores contemplados son:
    • Previo: en caso de seleccionar este valor, el nuevo campo se creará en una posición previa a un campo seleccionado.
    • Posterior: en caso de seleccionar este valor, el nuevo campo se creará en una posición posterior a un campo seleccionado. Esta posición es la fijada por defecto.
  • Mover: sección contiene los botones para mover un campo dentro de una línea. No se permite mover campos entre líneas. Las opciones de movimiento disponibles son:
    • Mover a la izquiera (<): permite mover hacia la izquierda un campo seleccionado. Este botón quedará deshabilitado cuando no exista la posibilidad de mover el componente más a la izquierda.
    • Mover a la derecha (>): permite mover hacia la derecha un campo seleccionado. Este botón quedará deshabilitado cuando no exista la posibilidad de mover el componente más a la derecha

EDITOR VISUAL

Este editor permite al desarrollador ver el resultado del formulario que se está creando en tiempo de diseño.
Los campos incluidos en este área son seleccionables, haciendo clic encima, de manera que en la zona derecha de la pantalla se presenten las propiedades del componente seleccionado para su edición
Los formulario diseñados se componen de páginas, y cada una de esas páginas deberán contener un conjunto de líneas sobre las que se situarán los componentes o campos.

PROPIEDADES DE COMPONENTE

  • Propiedades: en esta sección se configuran las propiedades del componente seleccionado.
    • Identificador: código únivoco que identifica al componente dentro del formulario. Este código se deberá utilizar cuando se quiera referenciar en campo en algun script, ya sea de formulario o de flujo de tramitación. Para mantener una convención respecto a los identificadores, se recomienda la utilización de mayúsculas.
    • Columnas ocupadas: número de columnas que ocupa el componente en una línea. Cada línea puede ocupar un máximo de 6 columnas.
    • Texto: texto multiidioma que se presentará como etiqueta del componente.
    • No mostrar texto: check que en caso de estar marcado hace que no se muestre etiqueta para el componente.
    • Ayuda en línea: texto multiidioma que se presentará como ayuda del componente al ciudadano. Este texto se mostrará en el frontal siempre y cuando el ciudadano mantenga las ayudas activas.
    • Obligatorio: indica si la cumplimentació del componente es requerida.
    • Sólo lectura: check que permite establecer el componente en modo sólo lectura, a nivel de HTML, de manera que el ciudadano no podrá introducir ningún valor en el mismo.
    • No modificable: check que permite establecer que se compruebe que un campo no var´a su valor respecto a los datos iniciales del formulario. Esta comprobación se realiza en servidor.
    • Tipo: esta opción está disponible únicamente para campos de texto y selector. Para un campo de texto permite fijar el tipo de valor esperado, de manera que se realicen unas validaciones acordes a dicho tipo (p.ej. Identificación permitiría validar si es un NIF de persona física: DNI, NIE u otros [NIF de tipo K, L y M]; jurídica: NIF; u otros: NSS). En el caso de un selector, permitirá fijar la manera en que se presentan los valores y si se posibilita selecionar uno o más valores.
    • Tipo lista valores (selector): esta opción está disponible únicamente para campos selector, y permite fijar de qu´ manera se alimentará los valores a mostrar en el campo. Las opciones disponibles son "Fija", en la que se deberán introducir los valores a mostrar, "Dominio", en la que se deberá seleccionar el dominio del que extraer los valores, y "Script", que requerirá la cumplimentación del script "Valores posibles" de la sección "Scripts".
  • Scripts: sección que presenta los scripts del campo. Los scripts disponibles son:
    • Autorelleno: script que permite asignar un valor al campo en función de valores de campos previos. Es importante que los campos de los que depende deben tener posiciones anteriores al campo sobre el que aplica el script.
    • Sólo lectura: script que permite asignar el estado "sólo lectura" al campo en función de valores de campos previos. Es importante que los campos de los que depende deben tener posiciones anteriores al campo sobre el que aplica el script.
    • Validación: script que permite realizar una validación del valor del campo. Este script permite tanto mostrar un mensaje informativo, permitiendo continuar con el trámite, o un error, de manera que no el ciudadano no pueda continuar con el trámite hasta satisfacer la validación.
    • Valores posibles: este script se muestra para campos selector con Tipo lista valores "Script", y permite establecer el conjunto de valores posibles del campo.
  • Estilos: permite modificar estilos del campo. Los estilos disponibles son:
    • Alineación texto: permite establecer la posición de la etiqueta de un campo.

BOTONADURA PIE DE PÁGINA

  • Paginación: permite cambiar de página, en caso de formulario multipágina, para la edición de componentes. La navegación permitida es:
    • Página anterior: permite ir a una página posterior del formulario, en caso de haberla, para la edición de la misma.
    • Página posterior: permite ir a una página posterior del formulario, en caso de haberla, para la edición de la misma.
  • Aplicar cambios: permite consolidar los cambios que se hayan realizado sobre un componente. Mientras no se apliquen los cambios, estos no serán visibles ni en el editor visual ni en el frontal.
  • Borrar componente: permite borrar un componente seleccionado en el editor visual.
  • Ver estructura: permite mostrar, en forma de árbol, la estructura de elementos del formulario (páginas, líneas y componentes).
  • Idioma: permite modificar el idioma a mostrar en el editor visual del formulario. Por defecto se visualiza en el mismo idioma que se está utilizando en la aplicación.
  • Previsualizar versión: permite acceder a la pantalla de configuración de previsualización de versión, a partir de la cual se podrá previsualizar la versión en desarrollo en el asistente de tramitación (vista del ciudadano).
  • Cerrar: cierra la pantalla de diseño del formulario.
