MANTENIMENTO DE PASO REGISTRAR FORMULARIO

En la pantalla de 'Mantenimento de paso Registrar formulario' puede gestionar las opciones relativas al registro de la solicitud.

CAMPOS

  • Datos de destino: en este apartado se han de consignar los datos de destino de las solicitudes vinculadas a la versión del trámite.
    • Oficina de registro: oficina del registro electrónico sobre el que se realizará el registro de la solicitud.
    • Libro de registro: libro de registro, de la oficina de registro seleccionada, sobre la que se realizará el apunte registral de entrada de la solicitud.
    • Tipos de asunto: tipo de asunto que se especificará al realizar el apunte registral sobre el registro electrónico.
    • Establecer datos destino: script que permite establecer dinámicamente los parámetros de destino descritos anteriormente. Además permite establecer el número de expediente (sería obligatorio para trámites que no son de inicio).
  • Instrucciones de presentación (HTML):Este concepto está en revisión, por lo que no deberí consignarse valores en dicho campo.
  • Instrucciones de tramitación (HTML): texto multiidioma, con soporte HTML, que permite consignar información de interés para el ciudadano que se mostrará en el asistente, en el paso Registrar.
  • Establecer presentador: script que permite indicar los datos de la persona física o jurídica que presenta la solicitud. Para tramitaciones autenticadas, si el script no tiene contenido, se tomará como presentador el usuario autenticado. Para tramitaciones no autenticadas será necesario consignar la persona presentadora.
  • Representación: configuraciones y restricciones relativas a la aportación electrónica del documento por parte del ciudadano:
    • Admite representación: check para indicar si el trámite permite representación. En caso de estar marcado, será necesaria la identificación de la persona representada en el script "Establecer representado."
    • Establecer representado: indica las extensiones de ficheros que se permitirán adjuntar para el documento anexo. Si se marca "Todas" se pemitirá cualquier extensión, y en caso de marcar "Personalizadas" se deberán consignar en el campo de texto las extensiones deseadas, separadas por punto y coma (;).
    • Valida representación: check que permite indicar si se ha de validar la representación, en caso de haberla. Esta opción no está actualmente disponible, ya que requiere del uso de un plugin de validación de representación.
  • Permite registrar: script que permite realizar validaciones previas al registro de una solicitud. Si dichas validaciones no se satisfacen, se podrí impedir el registro de la solicitud.

OPERATIVA

  • Guardar: cierra la pantalla guardando los cambios.
  • Cancelar: cierra la pantalla sin guardar los cambios.
