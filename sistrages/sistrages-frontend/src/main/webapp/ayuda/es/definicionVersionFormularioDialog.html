MANTENIMENTO DE FORMULARIO

En la pantalla de 'Mantenimiento de formulario' puede gestionar las configuraciones y diseño del formulario.

CAMPOS

  • Identificador: código unívoco que identifica al formulario en la aplicación. Cualquier referencia al formulario en los scripts de la versión se realizará utilizando este código.
  • Descripción: texto multiidioma de presentación del formulario en el asistente de tramitación. Esta será la denominación del formulario para el tramitador.
  • Obligatoriedad: permite configurar la obligatoriedad de cumplimentación del formulario en el asistente de tramitación. Los valores posibles son:
    • Obligatorio: si se marca esta opción será obligatoria la cumplimentación del formulario en el asistente. En caso contrario no se podrá con la cumplimentación del trámite.
    • Opcional: si se marca esta opción la cumplimentación del formulario será opcional.
    • Depende: si se marca esta opción la obligatoriedad del formulario dependerá del script 'Establecer dependencia'.
    • Establecer dependencia: script que permite establecer la obligatoriedad de un documento según otros datos del trámite (dominios, datos de un formulario previo, datos de sesió, etc.). únicamente se tendrá en cuenta el valor de este script si se ha fijado previamente la obligatoriedad 'Depende'.
  • Firmar digitalmente: permite establecer que se requiera la firma electrónica del formulario. Esta firma se solicitará en el último paso del asistente, de manera previa al registro del trámite.
  • Establecer firmante: script que permite, en caso de requerirse firma electrónica del fomulario, establecer el firmante o firmantes requeridos. Para tramitaciones autenticadas, si no se consignan firmantes, se solicitará la firma del usuario autenticado. Para tramitaciones no autenticadas será obligatorio consignar por lo menos un firmante en el script.
  • Script personalización: scripts que permiten llevar a cabo tareas de personalización del formulario. Los scripts de personalización disponibles son:
    • Carga de datos inicial: script que permite establecer valores inciales en los campos del formulario.
    • Parámetros de apertura del formulario: script que permite pasar parámetros a un formulario.
    • Postguardar formulario: script que permite tanto realizar validaciones tras guardar el formulario, como alimentar valores de otro formulario.
  • Editar diseño: permite, en caso de estar en modo edición, acceder al diseño del formulario. Si accedemos en modo consulta, el botón se presentará como 'Consultar diseño' y permitirá acceder al diseñador en modo consulta.

OPERATIVA

  • Guardar: cierra la pantalla guardando los cambios.
  • Cancelar: cierra la pantalla sin guardar los cambios.
