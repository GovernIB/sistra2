IMPORTACIÓN DE TRÁMITE/VERSIÓN

Esta funcionalidad de importación únicamente está disponible en entornos de desarrollo, y posibilita importar una versión de un trámite independientemente de cual sea su entorno de procedencia.
En esta pantalla se presentan las características del trámite/versión que se desea importar, indicando, en caso necesario, los aspectos a revisar y acciones a realizar por parte del usuario para poder finalizar satisfactoriamente el proceso de importación. En caso de requerirse una acción, se mostrará un botón "..." que presentará las acciones que podrá realizar el usuario para subsanar el warning presentado. Dado que cada versión de la plataforma implementa nuevas funcionalidades, es posible que exista alguna incompatibilidad entre versiones, por lo que es necesario que las versiones de los entornos origen y destino sean la misma. La pantalla presenta dos secciones diferenciadas, una titulada Versión trámite que presenta los datos asociados directamente a la versión (entidad, área, trámite y versión), y una segunda titulada Elementos comunes que presenta la información relativa a elementos comunes (dominios, formateadores, fuentes de datos, etc.) asociados a la versión que pueden ser comunes a otros trámites/versión.
Aspectos importantes del proceso de importación a tener en cuenta:

  • Limitaciones según rol de usuario: alguna de las acciones están limitada por el rol del usuario. Por ejemplo, en caso de importar un trámite proveniente de un área que no existe en el entorno, se dará la opción al usuario de crear un área nueva, pero únicamente lo podrá hacer si dispone del rol de Administrador de Entidad.
  • Desasociación de elementos: en determinadas situaciones se puede dar el caso de que un elemento asociado al trámite no pueda ser importado. Esto puede pasar tanto en dominios (p.ej. dominio de ámbito global no existente) como en formateadores (p.ej. utilización de formateador no existente). En este caso no se impedirá la importación de la versión del trámite, pero se desasociará el elemento afectado de la versión del trámite. Si el elemento afectado es el formateador, se eliminarán las plantillas de visualización definidas. Si el elemento afectado es un dominio, se eliminará de "Dominios empleados" y si es utilizado por un campo selector de un formulario, dicho selector pasará a ser de lista fija.

CAMPOS

  • Identificador: presenta el código identificador del elemento referenciado.
  • Descripción: presenta el texto descriptivo del elemento referenciado.
  • Ámbito: ámbito del elemento referenciado, puede ser Global, Entidad o Área.
  • Mensaje: muestra mensaje informativo, en caso necesario, respecto a los elementos del trámite/versión.
  • Acción: en caso de warning se presenta un botón que mostrará las posibles acciones a realizar por el usuario. Una vez ejecutada la acción, el elemento pasará a estado checked

ICONOGRAFIA

  • Info: este icono, representado con una exclamación, indica que se muestra alguna información de interés sobre el elemento a título informativo.
  • Warning: icono que indica que el elemento tiene algún aspecto a ser revisado por el usuario. En este caso se presenta un botón acción que permitirá realizar cambios que posibiliten que el elemento acabe como checked
  • Checked: icono que indica que el elemento está validado y listo para la importación.
  • Error: icono que indica un error relativo al elemento. Este estado podrá cambiar en función de las acciones que realice el usuario sobre elementos previos relacionados.
  • No importable: icono que indica que hay algún aspecto bloqueante para la importación. En este caso no se permitirá la finalización del proceso de importación del trámite/versión.

CASOS DE USO

Elemento Caso de uso Consideraciones
Entidad ENTIDAD DIFERENTE Se avisa al usuario que se está importando sobre una entidad diferente a la origen, pero no es una situación bloqueante, por lo que se permite continuar.
Área NO EXISTE En este caso se mostrará el botón de acción, pudiendo seleccionar un área existente sobre la que importar o la posibilida de crear un área nueva. El creación de una nueva área requerirá permisos de Administrador de Entidad. La importación sobre un área requerirá de permisos sobre dicha área.
EXISTE Si el área existe, se presenta la acción de importar sobre un área diferente, de entre las existentes. El usuario que está realizando la importación deberá disponer de permisos sobre el área destino.
Trámite NO EXISTE Si el trámite no existe, se da de alta automáticamente. Previo a este alta se permite cambiar el código del trámite.
EXISTE Si el trámite existe en el área sobre la que se está importando, se sobrescribirá la versión. En caso de que el trámite exista en un área diferente a la de destino, no se permitirá importar (no se pueden duplicar códigos de trámite). Se permite modificar el código del trámite a importar.
Versión NO EXISTE Si la versión no existe se permite crearla o reemplazar una existente, si la hubiera.
EXISTE Si la versión existe se posibilita realizar la acción de sobreescribir existente, generando nueva release, o cambiar el número de versión
Datos registro REVISAR De manera obligatoria se deberá realizar la acción de revisión de los parámetros de registro, pudiendo confirmar los de origen o estableciendo unos diferentes.
Dominio NO EXISTE Si el dominio es de ámbito global, no se importará el dominio y se eliminará la asociación a la versión del trámite (se quitará de dominios asociados a la versión y se los selectores de tipo dominio que lo utilicen pasarán a lista fija).
Si el dominio es de ámbito entidad, solo se podrá crear si el usuario es Administrador de Entidad. En caso que el usuario no sea Administrador de entidad, no se importará el dominio y se eliminará la asociación a la versión del trámite que se está importando.
Si no hay restricción de creación, se deberá revisar los datos del dominio y se importará
EXISTE Si el dominio existe pero no tiene el mismo ámbito se mostrará un mensaje informativo pero se podrá continuar con la importación
Si el dominio es de ámbito entidad o área, este deberá pertenecer a la entidad o área sobre la que se importa. En caso contrario se informará al usuario y se eliminará la asociación al dominio en la versión del trámite a importar.
Formateador NO EXISTE Si el formateador no existe se permitirá importar pero se eliminará la asociación al formatedor que haya en la versión importada (plantillas de visualización).
EXISTE Si el formateador existe en la entidad, se mantendrá la asociación. Si en la entidad destino el formateador existente está configurado como "Desactivar personalización", se eliminará la asociación del formateador al trámite.

OPERATIVA

  • Importar: Se llevará a cabo la importación de la versión. Este botón estará deshabilitado mientras haya acciones requeridas pendientes.
  • Cancelar: Cierra la pantalla de importación y cancela el proceso.
