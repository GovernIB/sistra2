MANTENIMENTO DOMINIO

En la pantalla de 'Matenimento dominio' puede mantener los datos correspondientes a un dominio.
Los campos obligatorios se presentan con un arterisco *.

CAMPOS

  • Identificador: código unívoco que identifica al dominio dentro de la aplicación. Este identificador se utilizarà en cualquier script de la aplicación para accedir al dominio
  • Descripción: texto descriptivo del objeto del dominio.
  • Configuración cacheo: el checkbox "Cachear dominio" permite establecer una caché referida a la invocació del dominio. Si se marca este checkbox se cacheará el resultado de una invocación durante un tiempo máximo de una hora, si no hay nuevas invocaciones, y se forzarà un refresco de la caché transcurridas 24 horas desde la primera invocación. En cualquier caso se puede forzar explicitamente el refresco de la caché mediante el botón Refrescar caché
  • Tipo: Tipología del dominio, que puede ser Consulta BBDD, Lista Fija, Consulta Remota o Fuente Datos, esta última no disponible en dominios de ámbit global. Según el tipo de dominio se tendrà que configurar:
    • Consulta BBDD: dominio que consulta datos de una base de datos:
      • JNDI Datasource: ha de contener el JNDI del datasource que contiene la BBDD que consultará el dominio. Se ha de tener en cuenta que este datasource ha de estar desplegado en el mismo servidor de aplicaciones que SISTRAGESBACK.
      • Query: consulta, en lenguaje SQL, a ejecutar sobre la BBDD indicada en el datasource. En caso que la consulta requiera parámetros estos se referenciarán mediante su código y precedido por el símbolo dos puntos (:). A continuación se presenta un ejemplo ilustrativo de una consulta con un parámetro identificador:

        SELECT ID, DESC1, DESC2 FROM TAULA WHERE ID = :identificador
    • Lista fija: dominio que recupera datos de una lista fija de valores. El mantenimento del listado de valores permite las operaciones siguientes:
      • Nuevo: permite crear un nuevo elemento en el listado. Este elemento consta del códi del elemento y del valor del mismo
      • Editar: permite editar un elemento ya existente en el listado. Antes de pulsar el botón de edición tendrá que seleccionar en el listado el elemento a editar.
      • Eliminar: permite eliminar un element existent al llistat. Abans de pitjar el botó d'eliminació haureu de seleccionar al llistat l'element a eliminar.
      • Subir: permite subir un elemento en el orden del listado. Para subir un elemento previamente ha de estar seleccionado en el listado.
      • Bajar: permite bajar un elemento en el orden del listado. Per subir un elemento previamente ha de estar seleccionado en el listado.
    • Consulta Remota: dominio que recupera valores de una ubicación remota a partir del plugin de dominio remoto:
      • URL: URL contra la que se hará la consulta remota.
    • Fuente Datos: dominio que recupera datos de una Fuente de Datos definida en SISTRAGESBACK:
      • Fuente: desplegable que permite seleccionar la fuente de datos a consultar. Únicamente se muestran las fuentes de datos que pertenecen al ámbito del dominio
      • Query: consulta, en lenguaje pseudoSQL para consultar una fuente de datos. El formato de esta consulta tiene que ser:

        SELECT campo (, campo)* FROM fuenteDatos WHERE expresion ( [AND | OR] expresion ] )* ORDER BY campo

        donde expresion: campo [= | LIKE] ?, donde "?" es un parámetro

        Las partículas de la sentencia pseudoSQL (SELECT, FROM, WHERE, LIKE, AND, OR, ORDER BY) han de estar en mayúsculas.
        Respecto a las comparanciones en el WHERE serán ignorecase ( = y LIKE ).
        Respecto al LIKE se realizará automáticamente la comparación: %parametro% .
        Respecto al ORDER BY solo se puede indicar un campo. Además como todos los campos son de tipo cadena, la ordenación es literal por lo que si se quiere crear un campo en la fuente de datos que permita ordenar secuencialmente, mejor utilizar letras que números.

  • Parámetros: listado de parámetros del dominio. Las operaciones que se pueden realizar sobre este listado son:
    • Nuevo: permite crear un parámetro. Este parámetro se identifica a partir de un código.
    • Editar: permite editar un parámetro ya existente en el listado. Antes de pulsar el botón de edición tendrá que seleccionar el parámetro a editar en el listado.
    • Eliminar: permite eliminar un parámetro existente en el listado. Antes de pulsar el botón de eliminación tendrá que seleccionar el parámetro a eliminar en el listado.
    • Subir: permite subir un parámetro en el orden del listado de parámetros. Per subir un parámetro previamente ha de estar seleccionado en el listado de parámetros.
    • Bajar: permite bajar un parámetro en el orden del listado de parámetros. Per bajar un parámetro previamente ha de estar seleccionado en el listado de parámetros.

OPERATIVA

  • Guardar: Cierra la pantalla guardando los cambios.
  • Cancelar: Cierra la pantalla sin guardar los cambios.
