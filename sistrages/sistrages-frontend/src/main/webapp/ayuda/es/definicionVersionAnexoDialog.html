MANTENIMENTO DE ANEXO

En la pantalla de 'Mantenimiento de anexo' puede gestionar las configuraciones del documento anexo.

CAMPOS

  • Identificador: código unívoco que identifica al anexo en la aplicación. Cualquier referencia al anexo en los scripts de la versión se realizará utilizando este código.
  • Descripción: texto multiidioma de presentación del anexo en el asistente de tramitación. Esta será la denominación del anexo que visualizará el tramitador en el asistente de tramitación.
  • Obligatoriedad: permite configurar la obligatoriedad de anexar el documento en el asistente de tramitación. Los valores posibles son:
    • Obligatorio: si se marca esta opción será obligatorio el anexado del documento en el asistente. En caso contrario no se podrá con la cumplimentación del trámite.
    • Opcional: si se marca esta opción el anexado del documento será opcional.
    • Depende: si se marca esta opción la obligatoriedad de anexado del documento dependerá del script 'Establecer dependencia'.
    • Establecer dependencia: script que permite establecer la obligatoriedad de un documento según otros datos del trámite (dominios, datos de un formulario previo, datos de sesió, etc.). únicamente se tendrá en cuenta el valor de este script si se ha fijado previamente la obligatoriedad 'Depende'.
  • Ayuda: en este apartado se configura la información relativa al documento anexo, que se mostrará al ciudadano en el asistente de tramitación para facilitar la aportación:
    • Fichero: esta opción permite al desarrollador adjuntar un documento que se mostrará en el asistente de tramitación a modo de plantilla del documento a anexar.
    • URL: permite indicar un enlace de la que el ciudadano pueda descargar, desde el asistente de tramitación, un documento a modo de plantilla del documento a anexar.
    • Mensaje (HTML): posibilita añadir un mensaje, con soporte HTML, que se mostrará en el asistente de tramitación como información de interés para el ciudadano para la aportación del documento.
  • Firmar digitalmente: permite establecer que se requiera la firma electrónica del formulario. Esta firma se solicitará en el último paso del asistente, de manera previa al registro del trámite.
  • Establecer firmante: script que permite, en caso de requerirse firma electrónica del fomulario, establecer el firmante o firmantes requeridos. Para tramitaciones autenticadas, si no se consignan firmantes, se solicitará la firma del usuario autenticado. Para tramitaciones no autenticadas será obligatorio consignar por lo menos un firmante en el script.
  • Presentación electrónica: configuraciones y restricciones relativas a la aportación electrónica del documento por parte del ciudadano:
    • Num. instancias: indica, en formato numérico entero, la cantidad de fichero que se permitirá adjuntar al ciudadano para el documento anexo.
    • Extensiones: indica las extensiones de ficheros que se permitirán adjuntar para el documento anexo. Si se marca "Todas" se pemitirá cualquier extensión, y en caso de marcar "Personalizadas" se deberán consignar en el campo de texto las extensiones deseadas, separadas por punto y coma (;).
    • Tam. máximo: indica, en formato numérico entero, el tamaño máximo de los ficheros que se permitirán adjuntar. Además dicho tamaño se puede expresar en kilobytes (KB) o megabytes (MB), marcando la opción correspondiente.
    • Convertir a PDF: indica si el fichero anexado se ha de convertir a PDF tras su aportación desde el asistente. Los formatos de ficheros convertibles dependerán del plugin de conversián a PDF definido en la instalación.
    • Anexar firmado: indica si el fichero aportado ha de ir firmado. En ese caso se validará la firma contenida en el documento. El formato del fichero a aportar está restringido a PDF.
    • Firmar: indica si el fichero aportado se deberá firmar electrónicamente desde el asistente de tramitación. En caso de estar marcada esta opción se solicitará la firma por parte del firmante o firmantes establecidos en el script "Establecer firmantes". Para tramitaciones autenticadas, si el script no tiene valor se solicitará la firma por parte del usuario autenticado.
    • Establecer firmantes: script que permite, en caso de requerirse firma electrónica del anexo, establecer el firmante o firmantes requeridos. Para tramitaciones autenticadas, si no se consignan firmantes, se solicitará la firma del usuario autenticado. Para tramitaciones no autenticadas será obligatorio consignar por lo menos un firmante en el script.
    • Script de validación: script que permite llevar a cabo validaciones tras la aportación del documento por parte del ciudadano. Este script se ejecuta al anexar el documento.

OPERATIVA

  • Guardar: cierra la pantalla guardando los cambios.
  • Cancelar: cierra la pantalla sin guardar los cambios.
