MANTENIMENTO ROLES Y PERMISOS

En la pantalla de 'Mantenimiento de roles y permisos' puede mantener los datos relativos a un rol/usuario.
Los campos obligatorios se presentan con un asterisco *

CAMPS

  • Tipo: permite indicar si se define un rol ("Rol") o un usuario ("Usuario").
  • Rol/Principal: este campo está relacionado con el campo "Tipo". Si se define un tipo "Rol", deberá contener el código del rol, y si por el contrario se define tipo "Usuario", deberá contener el código de usuario al que se otorgan los permisos.
  • Área: área sobre la que aplicarán los permisos definidos.
  • Descripción: texto descriptivo de los permisos definidos.
  • Permisos: permite seleccionar los permisos que se otorgan. Los permisos disponibles son:
    • Administrador: permite realizar acciones de mantenimiento sobre elementos comunes del área. También permitirá promocionar entre entornos y modificar las propiedades de acceso en los entornos de Preproducción y Producción.
    • Desarrollador: permite desarrollar trámites en el entorno de desarrollo para dicha área.
    • Consulta: permite consultar los elementos del área.
    • Helpdesk: permite realizar consulta desde el módulo SISTRAHELP sobre los eventos de trámites del área.

OPERATIVA

  • Guardar: cierra la pantalla guardando los cambios.
  • Cancelar: cierra la pantalla sin guardar los cambios.
