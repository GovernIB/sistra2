addParametroMensaje(String parametre): void

DESCRIPCIÓ

  • Mètode que permet establir el valor del paràmetre a substituir al missatge. Aquesta funcionalitat permet establir contingut dinàmic als missatges de validació.
  • Els missatges es defineixen a la pipella de missatges de validació amb un identificador.
  • Dins del missatge es defineixen els paràmetres mitjançant números entre claus, podent tenir-ne més d'un. P.ex. {0} {1}...
  • Si el missatge conté més d'un paràmetre, s'ha de tenir en compte que importa l'ordre dels paràmetres a l'hora de sustituir-los.

PARÀMETRES DE SORTIDA

  • parametre: String amb el valor del paràmetre a sustituir.

EXEMPLES

Posam per cas que volem fixar el valor dels paràmetres d'un missatge de validació, per exemple MENSAJE_001, prèviament creat. Les passes a seguir serien les següents:

En la pantalla de script es defineix, a la pipella de missatges de validació, el missatge amb identificador MENSAJE_001 i dos paràmetres mitjançant el text:

Es necessari que es dugui a terme un debug amb els paràmetres {0} y {1}.

Des del script es referencia el missatge de la manera següent:

var msg = PLUGIN_MENSAJES.crearMensaje('MENSAJE_001');

posteriorment se li afegeixen els valors pels paràmetres 0 i 1:

msg.addParametroMensaje('param 0'); msg.addParametroMensaje('param 1');

de manera que es substituirà les cadenes {0} i {1} per param0 i param1 respectivament.
