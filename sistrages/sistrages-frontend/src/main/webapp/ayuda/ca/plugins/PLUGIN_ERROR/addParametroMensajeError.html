addParametroMensajeError(String parametre):void

DESCRIPCIÓ

  • Mètode que permet afegir el valor dels paràmetres a substituir en el missatge d'error. D'aquesta manera es pot dotar de dinamisme al missatge d'error.
  • Els paràmetres s'identifiquen al missatge d'error mitjantçant claus ({0} {1} ...).
  • Si el missatge té més d'un paràmetre, s'ha de tenir present que importa l'ordre en que s'hi afegeixin els paràmetres.

PARÀMETRES D'ENTRADA

  • parametre: String amb el valor del paràmetre a fixar.

EXEMPLES

Posam per cas que es vol controlar la disponibilitat de dades per un determinat NIF a una data concreta. Aquesta operació es podria fer recuperant dades d'un domini i mostrant un missatge d'error en cas de no existir-hi dades. A continuació es descriu com s'hauria de procedir per parametritzar el missatge d'error.

Prèviament, a la pantalla de script s'ha de definir un missatge d'error a la pipella de missatges validació, per l'exemple amb l'identificador CodErrFechas i el valor següent:
El DNI {0} no té dades associades a data {1}.
Des del script, es fa referència a aquest missatge de manera següent:

PLUGIN_ERROR.setCodigoMensajeError('CodErrFechas');
PLUGIN_ERROR.addParametroMensajeError('12345678T');
PLUGIN_ERROR.addParametroMensajeError('10/09/2012');

Com a resultat de l'execució del script s'obtindria el resultat següent:
El DNI 12345678T té dades associades a data 10/09/2012.
